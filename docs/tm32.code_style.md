## File Organization

### File Header
Every source file must begin with a standardized header comment:

```c
/**
 * @file    filename.c
 * @author  Author Name <email@domain.com>
 * @date    YYYY/MM/DD
 *
 * @brief   Brief description of the file's purpose.
 */
```

### Include Order
1. System includes (`#include <stdio.h>`)
2. Project-specific includes (`#include <tm32cpu_common.h>`)
3. Local includes (same directory)

```c
/* Include Files **************************************************************/

#include <stdio.h>
#include <stdint.h>
#include <stdbool.h>

#include <tm32cpu_processor.h>
```

### Section Organization
Use consistent section headers with decorative comment lines:

```c
/* Private Structures *********************************************************/

/* Private Function Prototypes ************************************************/

/* Private Functions - Category Name *****************************************/

/* Public Functions ***********************************************************/
```

## Naming Conventions

### Prefix System
All public symbols must use the project prefix pattern: `TM32<PROJECT>_`

- **TM32CPU**: CPU project symbols
- **TM32ASM**: Assembler project symbols  
- **TM32LINK**: Linker project symbols
- **TM32EMU**: Emulator project symbols

### Functions
- **Public functions**: `TM32CPU_FunctionName`
- **Private functions**: `TM32CPU_FunctionName` (static)
- Use PascalCase for function names
- Be descriptive and specific

```c
// Public function
TM32CPU_API bool TM32CPU_InitializeProcessor (TM32CPU_Processor* processor);

// Private function  
static bool TM32CPU_SetErrorCode (TM32CPU_Processor* processor, uint8_t errorCode);
```

### Types and Structures
- **Structures**: `TM32CPU_StructureName`
- **Enumerations**: `TM32CPU_EnumName`
- **Enumeration values**: `TM32CPU_PREFIX_VALUE`
- **Type aliases**: `TM32CPU_TypeName`

```c
typedef struct TM32CPU_Processor TM32CPU_Processor;

typedef enum
{
    TM32CPU_RT_A    = 0b0000,   /** @brief `A` - 32-bit Accumulator */
    TM32CPU_RT_AW   = 0b0001,   /** @brief `AW` - 16-bit Accumulator */
} TM32CPU_RegisterType;
```

### Constants and Macros
- **Constants**: `TM32CPU_CONSTANT_NAME`
- **Macros**: `TM32CPU_MACRO_NAME`
- Use ALL_CAPS with underscores

```c
#define TM32CPU_INITIAL_PC              0x00003000
#define TM32CPU_STACK_LOWER             0xFFFE0003

#define TM32CPU_LogError(fmt, ...) \
    fprintf(stderr, "[%s | ERROR] " fmt "\n", __func__, ##__VA_ARGS__)
```

### Variables
- **Local variables**: `camelCase`
- **Structure members**: `camelCase`
- **Global variables**: Avoid when possible; if needed, use `TM32CPU_globalName`

```c
struct TM32CPU_Processor
{
    uint32_t    programCounter;     /** @brief `PC` - Program Counter */
    uint32_t    stackPointer;       /** @brief `SP` - Stack Pointer */
    bool        ime;                /** @brief `IME` - Interrupt Master Enable */
};
```

## Code Formatting

### Indentation
- Use **4 spaces** for indentation
- No tabs
- Align continuation lines consistently

### Braces
Use **Allman style** (braces on new lines):

```c
if (condition)
{
    // Code here
}

struct TM32CPU_Processor
{
    uint32_t member;
};

bool TM32CPU_Function ()
{
    return true;
}
```

### Line Length
- Maximum line length: **80 characters**
- Break long lines at logical points
- Align parameters in function calls

### Spacing
- One space after keywords: `if (`, `while (`, `for (`
- One space around operators: `a + b`, `x = y`
- One space between function names and opening parentheses: `function (args)`
- One space after commas: `arg1, arg2, arg3`

```c
if (processor->busReadCallback(address, &byte0) == false)
{
    return TM32CPU_SetErrorCode(processor, TM32CPU_EC_INVALID_READ);
}
```

### Alignment
Align similar elements vertically when beneficial:

```c
uint32_t    a;                  /** @brief `A` - Accumulator Register */
uint32_t    b;                  /** @brief `B` - General Purpose Register B */
uint32_t    c;                  /** @brief `C` - General Purpose Register C */
uint32_t    d;                  /** @brief `D` - General Purpose Register D */
```

```c
processor->busReadCallback(address,     &byte0) == false ||
processor->busReadCallback(address + 1, &byte1) == false ||
processor->busReadCallback(address + 2, &byte2) == false ||
processor->busReadCallback(address + 3, &byte3) == false
```

## Documentation Standards

### Function Documentation
Use Doxygen-style comments for all public functions:

```c
/**
 * @brief   Brief description of what the function does.
 * 
 * Longer description if needed, explaining the purpose,
 * behavior, and any important implementation details.
 * 
 * @param   paramName   Description of the parameter.
 * @param   processor   Points to the TM32CPU processor instance.
 * 
 * @return  Description of return value and conditions.
 */
TM32CPU_API bool TM32CPU_FunctionName (
    TM32CPU_Processor*  processor,
    uint32_t            paramName
);
```

### Inline Comments
- Use `//` for single-line comments
- Use `/* */` for multi-line explanations
- Place comments above the code they describe
- Align inline comments when documenting multiple related lines

```c
// Check if the read operation is within valid memory ranges
if (startAddress >= TM32CPU_INTERRUPT_START && endAddress <= TM32CPU_RAM_END)
{
    return true;
}

/*
    The `TM32CPU_Fetch_*` function names are an exception to the TM32 workspace's
    code style. These functions follow a specific naming convention to
    indicate their purpose and the types of operands they handle.
*/
```

### Structure Member Documentation
Document structure members inline:

```c
struct TM32CPU_Processor
{
    // Callback Functions
    TM32CPU_BusReadCallback   busReadCallback;
    TM32CPU_BusWriteCallback  busWriteCallback;
    TM32CPU_ClockCallback     clockCallback;

    // General Purpose Registers
    uint32_t    a;                  /** @brief `A` - Accumulator Register */
    uint32_t    b;                  /** @brief `B` - General Purpose Register B */
    
    // Special Purpose Registers
    uint32_t    pc;                 /** @brief `PC` - Program Counter */
    uint32_t    sp;                 /** @brief `SP` - Stack Pointer */
};
```

## Preprocessor Directives

### Header Guards
Use `#ifndef` style guards with project-specific naming:

```c
#ifndef TM32CPU_PROCESSOR_H
#define TM32CPU_PROCESSOR_H

// Content here

#endif // TM32CPU_PROCESSOR_H
```

### API Macros
Define platform-specific API export/import macros:

```c
#if !defined(TM32CPU_BUILD_STATIC)
    #if defined(TM32_WINDOWS)
        #ifdef TM32CPU_EXPORTING
            #define TM32CPU_API __declspec(dllexport)
        #else
            #define TM32CPU_API __declspec(dllimport)
        #endif
    #elif defined(TM32_LINUX) || defined(TM32_MACOS)
        #ifdef TM32CPU_EXPORTING
            #define TM32CPU_API __attribute__((visibility("default")))
        #else
            #define TM32CPU_API
        #endif
    #endif
#else
    #define TM32CPU_API
#endif
```

### Constants
Group related constants and provide documentation:

```c
// Memory Map Constants
#define TM32CPU_METADATA_START          0x00000000  /** @brief Start of the metadata region. */
#define TM32CPU_METADATA_END            0x00000FFF  /** @brief End of the metadata region. */
#define TM32CPU_PROGRAM_START           0x00003000  /** @brief Start of the program memory region. */
#define TM32CPU_PROGRAM_END             0x7FFFFFFF  /** @brief End of the program memory region. */
```

## Function Definitions

### Function Spacing
Always place a space between the function name and the opening parenthesis in both declarations and definitions:

```c
// Correct
TM32CPU_API bool TM32CPU_InitializeProcessor (TM32CPU_Processor* processor);
static void helperFunction (int param);

// Incorrect
TM32CPU_API bool TM32CPU_InitializeProcessor(TM32CPU_Processor* processor);
static void helperFunction(int param);
```

### Parameter Formatting
Align parameters vertically in function declarations:

```c
TM32CPU_API bool TM32CPU_ReadDoubleWord (
    TM32CPU_Processor*  processor,
    uint32_t            address,
    uint32_t*           data
);
```

For function definitions, keep the same alignment:

```c
bool TM32CPU_ReadDoubleWord (
    TM32CPU_Processor*  processor,
    uint32_t            address,
    uint32_t*           data
)
{
    // Implementation
}
```

### Return Statements
- Use explicit parentheses for complex return expressions
- Prefer early returns for error conditions
- Use consistent error handling patterns

```c
if (processor == NULL)
{
    return false;
}

return TM32CPU_ConsumeCycles(processor, 4);
```

## Variable Declarations

### Initialization
- Initialize variables at declaration when possible
- Group related variable declarations
- Align variable names when beneficial

```c
uint8_t byte0 = 0, 
        byte1 = 0, 
        byte2 = 0, 
        byte3 = 0;

uint64_t    startAddress = (uint64_t) address,
            endAddress   = (uint64_t) address + (uint64_t) readSize - 1;
```

### Pointer Declarations
Attach the asterisk to the type:

```c
TM32CPU_Processor*  processor;
uint32_t*           data;
const char*         message;
```

## Control Flow

### Conditional Statements
Always use braces, even for single statements:

```c
if (condition)
{
    singleStatement ();
}

if (condition1)
{
    // First case
}
else if (condition2)
{
    // Second case
}
else
{
    // Default case
}
```

### Switch Statements
```c
switch (processor->iparam1 & 0b11)
{
case 0b00: // Full 32-bit register
    return 
        TM32CPU_ReadDoubleWord(processor, processor->pc, &processor->sd) &&
        TM32CPU_AdvanceProgramCounter(processor, 4);
case 0b01: // Partial 16-bit register
    return 
        TM32CPU_ReadWord(processor, processor->pc, (uint16_t*) &processor->sd) &&
        TM32CPU_AdvanceProgramCounter(processor, 2);
default: // Partial 8-bit register
    return 
        TM32CPU_ReadByte(processor, processor->pc, (uint8_t*) &processor->sd) &&
        TM32CPU_AdvanceProgramCounter(processor, 1);
}
```

### Loops
```c
for (uint32_t i = 0; i < count; i++)
{
    // Loop body
}

while (condition)
{
    // Loop body
}
```

## Error Handling

### Error Checking Patterns
Use the project's error handling macros:

```c
TM32CPU_ReturnValueIfNull(processor, false);
TM32CPU_ReturnValueIf(address > MAX_ADDRESS, false);
```

### Function Error Returns
- Return `bool` for success/failure operations
- Return `NULL` for pointer-returning functions on error
- Use consistent error codes when applicable

```c
if (TM32CPU_ReadByte(processor, address, &data) == false)
{
    return TM32CPU_SetErrorCode(processor, TM32CPU_EC_INVALID_READ);
}
```

### Logging
Use project-specific logging macros:

```c
TM32CPU_LogError("Failed to allocate memory for processor");
TM32CPU_LogWarn("Condition failed: %s", #clause);
TM32CPU_LogInfo("Processor initialized successfully");
```

## Memory Management

### Allocation Macros
Use project-specific memory management macros:

```c
TM32CPU_Processor* processor = TM32CPU_Create(1, TM32CPU_Processor);
TM32CPU_Processor* array = TM32CPU_CreateZero(count, TM32CPU_Processor);
TM32CPU_Destroy(processor);
```

### Memory Safety
- Always initialize pointers (to `NULL` or valid memory).
- Always check allocation results
- Set pointers to `NULL` after freeing
- Use the `TM32CPU_Destroy` macro which handles NULL checks

## Header Files

### Structure
1. File header comment
2. Header guard start
3. Include files section
4. Forward declarations and typedefs
5. Public constants
6. Public enumerations
7. Public function prototypes
8. Header guard end

### Forward Declarations
Prefer forward declarations in headers when possible:

```c
/**
 * @brief   Contains the TM32CPU Processor structure.
 */
typedef struct TM32CPU_Processor TM32CPU_Processor;
```

### Function Prototypes
Group related function prototypes and document their purpose:

```c
/**
 * @brief   Allocates and creates a new TM32CPU Processor instance.
 */
TM32CPU_API TM32CPU_Processor* TM32CPU_CreateProcessor (
    TM32CPU_BusReadCallback   busReadCallback,
    TM32CPU_BusWriteCallback  busWriteCallback,
    TM32CPU_ClockCallback     clockCallback
);
```

## Project-Specific Conventions

### Exception Cases
Some function naming patterns are exceptions to the general rules:

1. **Fetch Functions**: `TM32CPU_Fetch_<DEST>_<SRC>` pattern for instruction operand fetching
2. **Execute Functions**: `TM32CPU_Execute_<INSTRUCTION>` pattern for instruction execution

### Binary Constants
Use binary notation for bit patterns when it enhances clarity:

```c
TM32CPU_RT_A    = 0b0000,   /** @brief `A` - 32-bit Accumulator */
TM32CPU_RT_AW   = 0b0001,   /** @brief `AW` - 16-bit Accumulator */
```

### Register Size Checking
Use consistent bit patterns for register size validation:

```c
switch (processor->iparam1 & 0b11)
{
case 0b00: // Full 32-bit register
case 0b01: // Partial 16-bit register  
default:   // Partial 8-bit register
}
```

### Flag Operations
Use descriptive function names for flag operations:

```c
TM32CPU_SetFlags(processor, TM32CPU_FT_C);
TM32CPU_ClearFlags(processor, TM32CPU_FT_H | TM32CPU_FT_N);
TM32CPU_ChangeFlags(processor, TM32CPU_FT_Z, result == 0);
TM32CPU_ToggleFlags(processor, TM32CPU_FT_C);
```

## Conclusion

Adherence to these coding standards ensures that all TM32 workspace code is consistent, readable, and maintainable. When in doubt, refer to existing code in the `tm32cpu` project as the reference implementation of these guidelines.

For questions or clarifications about these standards, refer to the existing codebase or consult with the project maintainers.
